This document was originally compiled in RMarkdown then converted into Quarto (a different flavor of markdown). It now tracks the SCW16 model sequence for Chilean jack mackerel (Trachurus murphyi), beginning with the documented 1.14 model series and moving to the candidate 0.05 pair for now. The files associated with this document can be found on Github.

The code to create the input files for this assessment and to run these models can be found in jjm/assessment/R/SC13_Bridging.R. Should you choose to run the models, please ensure that you have:

  • Compiled the latest jjm/src/jjms.tpl
    • This can be done by navigating to the jjm/src folder in your Terminal, and using the make command
    • You will need to have ADMB installed to do this.
    • This does not need to be done if you’re only re-compiling the R Markdown document.
  • Updated the jjmR package
    • This can be done in R using the command remotes::install_github("SPRFMO/jjmR")

fn_seldata <- function(x, i) {
  x$data$Index[,i] %>%
    bind_rows() %>%
    pivot_longer(everything(), names_to = "year") %>%
    drop_na() %>%
    mutate(
      year = as.numeric(year),
      assessment_year = ifelse(
        i==7,
        max(year)+1,
        max(year)
      )
    )
} 

fn_plotind <- function(mods2compare, indname) {
  mods <- compareModels(geth(mods2compare, "h1"))
  i <- grep(indname,mods[[1]]$data$Inames)
  dat2use <- list()
  for(m in 1:length(mods)) {
    dat2use[[m]] <- fn_seldata(mods[[m]],i)
  }

  p <- map_dfr(dat2use, ~as_tibble(.)) %>%
    mutate(assessment_year=as.factor(assessment_year)) %>%
    ggplot() +
      geom_line(aes(x=year,y=value,colour=assessment_year)) +
      scale_x_continuous(breaks= scales::pretty_breaks())
  print(p)
}

model_file_status <- function(model_name) {
  tibble(
    model = model_name,
    control_file = file.exists(file.path("config", paste0(model_name, ".ctl"))),
    data_file = file.exists(file.path("input", paste0(str_remove(model_name, "^h[12]_"), ".dat"))),
    report_file = file.exists(file.path("results", paste0(model_name, ".rep"))),
    yield_file = file.exists(file.path("results", paste0(model_name, ".yld")))
  )
}

1 Model Naming Convention

File naming conventions have been changed to reflect the stock structure hypotheses associated with each run. The h1 denotes the single-stock hypothesis, while h2 denotes the two-stock one.

For SCW16, the work is being reorganized around the reference pair and the paired 0.xx data-replacement sequence:

  • h1_1.14 and h2_1.14 are the documented reference models from the SC13 update sequence. They provide the immediate baseline for biological inputs, index treatment, selectivity changes, and FarNorth weight-at-age corrections.
  • h1_0.00 and h2_0.00 relabel the 1.14 reference pair as the SCW16 data-update baseline. The 0.00.dat file is the 1.14.dat base data file, so the h1_1.14 and h2_1.14 outputs remain the reference output pair for the first comparisons until the relabeled 0.00 controls are explicitly rerun.
  • h1_0.01 and h2_0.01 replace the South-Central Chile fishery CPUE (Chile_CPUE) with the 1998-2025 standardized series using weighted CVs.
  • h1_0.02 and h2_0.02 replace the FarNorth-associated CPUE (Peru_CPUE) with the 2015-2025 series after the Chile CPUE replacement.
  • h1_0.03 and h2_0.03 replace the FarNorth fishery length-composition data with the 1972-2025 series after the FarNorth-associated CPUE replacement.
  • h1_0.04 and h2_0.04 remove the Peru acoustic index from the 0.03 data set while retaining the new 2015-2025 FarNorth-associated CPUE series as Peru_CPUE.
  • h1_0.05 and h2_0.05 add the split Peru acoustic seasonal survey indices as Peru_Acoust_S1 and Peru_Acoust_S2, with length-composition data and independent survey selectivity blocks for each new index. This is the active SCW16 candidate pair for now.
