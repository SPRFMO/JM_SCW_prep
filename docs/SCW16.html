This document was originally compiled in RMarkdown then converted into Quarto (a different flavor of markdown). It shows the incremental results of the Chilean Jack Mackerel (Trachurus murphyi) stock assessment update in 2026. The files associated with this document can be found on Github.

The code to create the input files for this assessment and to run these models can be found in jjm/assessment/R/SC13_Bridging.R. Should you choose to run the models, please ensure that you have:

  • Compiled the latest jjm/src/jjms.tpl
    • This can be done by navigating to the jjm/src folder in your Terminal, and using the make command
    • You will need to have ADMB installed to do this.
    • This does not need to be done if you’re only re-compiling the R Markdown document.
  • Updated the jjmR package
    • This can be done in R using the command remotes::install_github("SPRFMO/jjmR")

fn_seldata <- function(x, i) {
  i_offshore <- grep("Offshore_CPUE", x$data$Inames)
  x$data$Index[,i] %>%
    bind_rows() %>%
    pivot_longer(everything(), names_to = "year") %>%
    drop_na() %>%
    mutate(
      year = as.numeric(year),
      assessment_year = ifelse(
        i==i_offshore,
        max(year)+1,
        max(year)
      )
    )
} 

fn_plotind <- function(mods2compare, indname) {
  mods <- compareModels(geth(mods2compare, "h1"))
  i <- grep(indname,mods[[1]]$data$Inames)
  dat2use <- list()
  for(m in 1:length(mods)) {
    dat2use[[m]] <- fn_seldata(mods[[m]],i) %>%
      mutate(assessment_year = ifelse(
        m == 1, yr_prev, yr_curr
      ))
  }

  p <- map_dfr(dat2use, ~as_tibble(.)) %>%
    mutate(assessment_year=as.factor(assessment_year)) %>%
    ggplot() +
      geom_line(aes(x=year,y=value,colour=assessment_year)) +
      scale_x_continuous(breaks= scales::pretty_breaks())
  print(p)
}

1 Model Naming Convention

File naming conventions have been changed to reflect the stock structure hypotheses associated with each run. The h1 denotes the single-stock hypothesis, while h2 denotes the two-stock one.

This working draft includes the single-stock (h1) and analogous two-stock (h2) comparisons for the 0.00 baseline and 0.11 bridge configuration.

The existing model numbers are retained to match the input, configuration, and result files. The table below lists the decision-level model configurations used in this document. Intermediate models 0.01 through 0.10 were retained as audit steps for sequential data introductions and corrections needed to reach the meeting specifications, but they are not treated here as separate model configurations.

Model Description
Models 0.x Bridging configurations
0.00 Baseline 2025 SC13 framing for the single-stock h1 and two-stock h2 hypotheses, retained with 0.00 naming for the bridge comparisons.
0.11 Current bridge from 0.00, including the updated data, specified input corrections, Chilean CPUE and Peru acoustic-survey catchability settings, and rescaled sample sizes for the Peru acoustic-survey length compositions.
Model Sensitivity Analyses
1.01 As 0.11 but with the full Chilean CPUE series from the previous assessment.
1.02 As 0.11 but with the full Chilean CPUE series from the previous assessment and the alternative sensitivity settings documented in the control file.

You’ll need to be in the jjm/assessment directory in order for the code here to run.

3 Bridging Model

The key bridging comparison is model 0.11 relative to model 0.00. Model 0.00 provides the baseline SC13 framing for both stock-structure hypotheses, while model 0.11 represents the cumulative update to the meeting specifications.

The intermediate runs from 0.01 through 0.10 introduced and checked individual data and configuration changes, including the revised Chilean and Peruvian CPUE inputs, Peruvian fishery length-composition updates, Peru acoustic-survey inputs and length compositions, sample-size and index-error rescaling, effort-creep assumptions, and catchability-break specifications. These runs document the path to 0.11, but are not shown as separate model configurations here.
