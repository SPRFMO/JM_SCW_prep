    • This does not need to be done if you’re only re-compiling the R Markdown document.

fn_seldata <- function(x, i) {
  i_offshore <- grep("Offshore_CPUE", x$data$Inames)
  x$data$Index[,i] %>%
    bind_rows() %>%
    pivot_longer(everything(), names_to = "year") %>%
    drop_na() %>%
    mutate(
      year = as.numeric(year),
      assessment_year = ifelse(
        i==i_offshore,
        max(year)+1,
        max(year)
      )
    )
} 

fn_plotind <- function(mods2compare, indname) {
  mods <- compareModels(geth(mods2compare, "h1"))
  i <- grep(indname,mods[[1]]$data$Inames)
  dat2use <- list()
  for(m in 1:length(mods)) {
    dat2use[[m]] <- fn_seldata(mods[[m]],i) %>%
      mutate(assessment_year = ifelse(
        m == 1, yr_prev, yr_curr
      ))
  }

  p <- map_dfr(dat2use, ~as_tibble(.)) %>%
    mutate(assessment_year=as.factor(assessment_year)) %>%
    ggplot() +
      geom_line(aes(x=year,y=value,colour=assessment_year)) +
      scale_x_continuous(breaks= scales::pretty_breaks())
  print(p)
}

plot_index_fit_compare <- function(res_tidy, base_model, focal_indices) {
  index_fits <- res_tidy$index_fits %>%
    filter(fleet_name %in% focal_indices)

  join_cols <- intersect(
    c("stock", "stock_name", "fleet_name", "year"),
    names(index_fits)
  )

  base_obs <- index_fits %>%
    filter(model == base_model) %>%
    mutate(
      low = observed_ind - 1.96 * observed_se,
      upp = observed_ind + 1.96 * observed_se
    ) %>%
    select(
      all_of(join_cols),
      observed_ind_base = observed_ind,
      low,
      upp
    )

  index_fits %>%
    left_join(base_obs, by = join_cols) %>%
    ggplot() +
      geom_pointrange(
        aes(year, observed_ind_base, ymin = low, ymax = upp),
        alpha = 0.45,
        colour = "grey35"
      ) +
      geom_path(aes(year, pred_ind, colour = model)) +
      scale_x_continuous(name = "Year") +
      scale_y_continuous(name = "Index Values") +
      facet_wrap(vars(fleet_name), scales = "free_y")
}

relabel_models <- function(mods, labels) {
  current_names <- names(mods)
  new_names <- unname(labels[current_names])
  names(mods) <- ifelse(is.na(new_names), current_names, new_names)
  mods
}

1 Model Naming Convention

File naming conventions have been changed to reflect the stock structure hypotheses associated with each run. The h1 denotes the single-stock hypothesis, while h2 denotes the two-stock one.

This working draft includes the single-stock (h1) and analogous two-stock (h2) comparisons for the 0.00 baseline and 0.11 bridge configuration.

The existing model numbers are retained to match the input, configuration, and result files. The table below lists the decision-level model configurations used in this document. Intermediate models 0.01 through 0.10 were retained as audit steps for sequential data introductions and corrections needed to reach the meeting specifications, but they are not treated here as separate model configurations.

3 Bridging Model

The key bridging comparison is model 0.11 relative to model 0.00. Model 0.00 provides the baseline SC13 framing for both stock-structure hypotheses, while model 0.11 represents the cumulative update to the meeting specifications.

The intermediate runs from 0.01 through 0.10 introduced and checked individual data and configuration changes. These included the revised Chilean and Peruvian CPUE inputs, Peruvian fishery length-composition updates, Peru acoustic-survey inputs and length compositions, sample-size and index-error rescaling, effort-creep assumptions, and catchability-break specifications. Unfortunately, because these specific runs (and the numbering/naming of them) were created quickly during the meeting, they included several mis-specifications and required correcting. This meant that while tracked, the usual “add a new piece of information” process ended up being done for only partially correct specifications. For this reason, we examined the effect of new information in the subsequent models where we had better control on models that were specified as intended by the working group. For these reasons, we omit presenting the separate model configurations (0.01-0.10) here.

4.3 Two-stock FarNorth fishery selectivity sensitivity

Model 0.12 is a two-stock-only sensitivity based on h2_0.11. It retains the 0.11 data and other h2 control-file settings, but replaces the two-period FarNorth fishery selectivity block in h2_0.11 with the annual time-varying FarNorth selectivity specification used in h1_0.11. This one-off sensitivity isolates whether the simpler two-period FarNorth fishery selectivity treatment affects the two-stock results.

4.4 Peruvian CPUE catchability-break sensitivity

Model 0.13 is based on 0.11 and retains the same data, selectivity, and weighting specifications, but adds a catchability break in 2018 for the Peruvian CPUE abundance-index series. This sensitivity evaluates whether allowing a step change in Peruvian CPUE catchability accounts for recent changes in that series and whether doing so materially changes stock-status and fishing-mortality patterns relative to 0.11.

4.5 Downweighting the Chile acoustic north and Peruvian CPUE indices

Model 1.15 starts from the 0.00 data and control-file settings but downweights the Chile acoustic north (Chile_AcousN) and Peruvian CPUE (Peru_CPUE) abundance-index series by inflating their index errors by 100x. These two series were pessimistic in the previous assessment cycle, so the rationale is to evaluate whether removing their fitting influence from the 0.00 configuration produces stock-status and fishing-mortality patterns similar to the full new-data fit in 0.11. The comparison with 0.11 therefore helps separate patterns associated with the new data from patterns associated with the influence of these two prior-year series. Because the observations are retained but given negligible fitting weight, 1.15 is treated as a diagnostic sensitivity rather than an alternative base-model candidate.
